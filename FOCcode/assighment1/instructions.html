In this assignment you will solve 6 problems that assess your skills in dealing with Ubuntu, Output, Input, and Selection in C.

Each program will be written in a different file, but all programs will be submitted together in one compressed file called handin.tar.

Read the following instructions completely before you start writing code to understand what the problems are, where to solve them, how to prepare your handin.tar submission file, and how to test it before submitting.

Getting Started

  1. Download the lab materials from Autolab using the "Download handout" link and extract its content. You will get one directory called "assignment1-handout".
  2. Inside the assignment1-handout directory, you will find a few files. Check the next section to understand what each file does.
  3. You need to install a program called make to help you test your code and prepare your submissions.

    4. You only have to install make once, so you will not need to do it again in the next assignments.

    To install make on Ubuntu, run the following command in the Ubuntu terminal and write your password when prompted to accept the installation:

    sudo apt install make

    Note for Mac users: If you are using Mac, you probably have make installed already. If not, you need to install the Mac Command Line Tools.

Assignment Files

Inside the extracted directory, you will find multiple C files, 1 file called problem1.txt, 3 files called checker.sh, runguard.c, and Makefile, and a directory called tests. You will need to modify some of these files, while some others should not be modified at all.

Files you need to modify:

  • Problem files:

    • Each problem will have one file in this assignment. Problem 1 will be solved in a text file called problem1.txt but other problems have C files called problem2.c for problem 2, and problem3.c for problem 3, and so on. The file of each problem will be used to solve only that problem.

    When you submit your assignment, all the problem files will be submitted, even if they are empty. Do not worry, you will only receive grades for the files that you implement, any unimplemented file will get 0 points until it is implemented and graded again. Do NOT delete any of the problem files or change their names. If you do that, they will not be graded.

    To understand what you need to do in each problem, continue reading. Each problem is explained in the Problems section below.

Files you should NOT modify:

  • The tests directory and anything inside it:

    • This directory contains a number of sample tests that will be used to test your code before you submit it. Each problem has its own directory inside tests, and inside each problem's directory there are two directories: inputs and outputs. The inputs directory contains the inputs of the test cases, and the outputs directory contains the expected outputs for the inputs. Do not move, remove, or modify these files. If you do that, you will not be able to test your solutions on your machine.

  • checker.sh and runguard.c:

    • These two files will help you grade most of your work on your laptop without having to submit it to Autolab. You will not use these file by yourself, but you will be using a command that uses them, so do not worry about them. You do not need to open them or understand how they work.

  • Makefile:

    • This file contains a few commands (called "make commands") that will make your life A LOT EASIER if you are using Ubuntu or Mac. Submitting your Work

When you submit your code to Autolab, we test it using a combination of tests. First we apply the tests that were available to you by running the testing make command, and second by running another group of tests that are not available to you. This means that your code might get a full grade on your laptop but might get a lower grade on Autolab because it fails a few other tests. Only your grade on Autolab will be counted.

To submit your work to Autolab:

  1. Create a handin.tar file that contains your solutions as described in the section above.
  2. Upload handin.tar to Autolab in the assignment page.
  3. Accept the academic integrity compliance check.
  4. Click submit.

Important note: You can only make 7 total submissions for this assignment. Only your last submission will be counted toward your grade. The website displays the number of remaining submissions you have under the assignment submission page.

Before submitting your work to Autolab, make sure that you test your code by running the testing make command, and by yourself (compile it with gcc and try it yourself) if you want to try other cases. You can test your work on your laptop as many times as you want before you submit it to Autolab. But you can only submit it 7 times to Autolab for this assignment.

Problem 6

Solve this problem inside the file problem6.c.

Write a program that accepts 3 different integers as inputs that represent a date in the form of: day month year (for example: 1 1 2023), then calculates which day of the year the date represents.

The day of the year is calculated by adding the current day of the month to the number of days in the months that passed.

Accept the input WITHOUT printing any sentences like "Please enter the date". If you print anything extra it will be considered part of the answer and will make your answer wrong.

Examples

Input Output
1 1 2023 1
4 7 2021 185
12 3 1981 71
5 12 2020 340
14 12 1800 348

Notes:

  • February has 28 days, unless the year is a leap year, then it has 29 days.
  • Leap years are divisible by 4, except for century years (years that end with 00). Century years are leap years only if they are divisible by 400.
